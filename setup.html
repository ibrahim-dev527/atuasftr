Create a Supabase Account & Project

Supabase is your free database — it stores all student registrations.

💡
Supabase is completely free for small projects. You don't need a credit card. The free plan supports up to 500MB of data and 50,000 API requests per month — more than enough for this trip registration.

Create the Database Table

Run one SQL command to create the registrations table and set permissions.

ℹ️
What is SQL? It's a command language for databases. You just need to copy and paste the code below — you don't need to understand it.
How to open the SQL Editor in Supabase
  1. 1 In your Supabase dashboard, look at the left sidebar and click "SQL Editor" (it looks like </>)
  2. 2 Click "New query" to open a blank editor
  3. 3 Copy the entire SQL block below and paste it into the editor
  4. 4 Click the green "Run" button (or press Ctrl+Enter)
  5. 5 You should see "Success. No rows returned" — that means it worked! ✅
SQL — Copy this entire block 
-- Step 1: Create the registrations table
CREATE TABLE trip_registrations (
  id              uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  full_name       text NOT NULL,
  index_number    text NOT NULL,
  contact_number  text NOT NULL,
  gender          text NOT NULL,
  availability    text NOT NULL,
  created_at      timestamptz DEFAULT now()
);

-- Step 2: Enable Row Level Security
ALTER TABLE trip_registrations ENABLE ROW LEVEL SECURITY;

-- Step 3: Allow anyone to read registrations (for admin)
CREATE POLICY "anon_select" ON trip_registrations
  FOR SELECT USING (true);

-- Step 4: Allow students to submit registrations
CREATE POLICY "anon_insert" ON trip_registrations
  FOR INSERT WITH CHECK (true);

-- Step 5: Allow admin to delete registrations
CREATE POLICY "anon_delete" ON trip_registrations
  FOR DELETE USING (true);

-- Step 6: Enable real-time updates
ALTER PUBLICATION supabase_realtime ADD TABLE trip_registrations;
⚠️
Copy the ENTIRE block — from CREATE TABLE all the way down to the last line. If you only copy part of it, the website won't work correctly.

Get Your API Keys

Two secret values that connect your website to the database.

🔑
What are API keys? Think of them like a username and password that your website uses to talk to the database. You'll copy them and paste them into your code.
Where to find your keys
  1. 1 In the Supabase sidebar, click the ⚙️ Settings icon (at the very bottom)
  2. 2 In the Settings menu, click "API"
  3. 3 You will see two important values — copy them both:

    📌 Project URL — looks like:
    https://xyzabcdef.supabase.co

    📌 anon public key — a very long string starting with eyJ...
  4. 4 Paste both values into a Notepad or text file so you have them ready for Step 4
⚠️
Use the "anon public" key only — NOT the "service_role" key. The service key is dangerous to expose. The anon key is the safe one.

Configure supabase.js

Replace the placeholder text with your real API keys.

How to edit supabase.js
  1. 1 Open the project folder you downloaded (the ZIP file — extract it first)
  2. 2 Find and open the file called supabase.js in any text editor
    (Notepad, VS Code, Sublime Text — any works)
  3. 3 You will see these two lines at the top. Replace the placeholder text with your real keys from Step 3:

❌ BEFORE (what it looks like now):

supabase.js — Current (placeholder) 
const SUPABASE_URL      = 'YOUR_SUPABASE_URL';
const SUPABASE_ANON_KEY = 'YOUR_SUPABASE_ANON_KEY';

✅ AFTER (what it should look like with your real keys):

supabase.js — After editing (example) 
const SUPABASE_URL      = 'https://xyzabcdef.supabase.co';  // ← paste your Project URL
const SUPABASE_ANON_KEY = 'eyJhbGciOiJIUzI1NiIsInR5c...';  // ← paste your anon key
✅
Keep the quotes! Make sure your URL and key are still inside the single quote marks '...'. Don't delete them.
After editing, save the file

Press Ctrl + S (Windows) or Cmd + S (Mac) to save the file. That's it — both index.html and admin.html share this same supabase.js file, so you only need to edit it once.

Option A — Deploy with Vercel (Recommended · 60 seconds)
  1. 1 Go to vercel.com and sign up (free) with GitHub or email
  2. 2 Click "Add New → Project"
  3. 3 Click "Upload" and drag your entire atuas-trip folder onto the page
  4. 4 Click "Deploy" — wait about 30 seconds
  5. 5 Vercel gives you a link like atuas-trip.vercel.app — that's your live website! 🎉
Option B — Deploy with Netlify (Drag & Drop)
  1. 1 Go to app.netlify.com/drop — no account needed!
  2. 2 Drag your entire atuas-trip folder onto the big drop area
  3. 3 Your site will be live instantly with a random URL. Sign up to keep it and get a custom name.
Option C — GitHub Pages (Free forever)
  1. 1Create a free account at github.com
  2. 2Click "New repository", name it atuas-trip, make it Public, click Create
  3. 3Upload all your project files using the "uploading an existing file" link
  4. 4Go to Settings → Pages, set Branch to main, folder to / (root), click Save
  5. 5Wait 2 minutes. Your site will be at yourusername.github.io/atuas-trip
